Baseline Vs Progressive images

Here I have save two large images to compare their loading. Note the actually performance benefits of progressive loading are negligible.

Both images are exactly the same size and no other optimizations have been made aside save for web.

To really notice the loading difference you may have to throttle the network.

Baseline (1254kb)
Progressive (1205kb)
Srcset using pixel density (1x,2x,3x etc)
Srcset using width (301w, 602w)
Srcset using width (302w, 604w) - with sizes attribute set to 302px
Mobile-first picture element - not serving images to mobile (will need to be display none)

An image at 2x will not download until pixel density is at least 50% higher than the closest declared pixel density (1x, 2x, 3x, 4x etc).

If pixel denstity is say 1.4x of the standard image, it will show the standard image. This is different to the original retina script which will fetch the 2x image the moment the pixel density was any amount above the standard image.

Once a higher resolution has been downloaded it will be used regardess of screen size unless stated otherwise, this is a performance feature because of catching, it makes no senes to fetch the "correct image" when you have a higher DPI version still cached
